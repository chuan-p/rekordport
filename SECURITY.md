# Security Policy

## Supported Versions

Security fixes are handled on a best-effort basis for the latest code and the
most recent release line.

| Version | Status |
| --- | --- |
| `main` | Supported |
| Latest GitHub release | Supported |
| Older releases | Not supported |

## Reporting A Vulnerability

Please avoid opening a public issue for vulnerabilities that could put users or
their libraries at risk.

Preferred path:

1. Use GitHub's private vulnerability reporting for this repository if the
   "Report a vulnerability" option is available.
2. If private reporting is not available, open a minimal public issue without
   exploit details and ask for a private follow-up channel.

Useful details to include:

- Affected version or commit
- Platform and OS version
- Whether the issue requires local access or a crafted media/database file
- Reproduction steps or a proof of concept
- Impact, especially if the issue can corrupt a Rekordbox library or leak data

## Response Expectations

- Initial acknowledgement: within 7 days
- Follow-up status update: within 14 days when reproduction is possible
- Fix target: latest supported branch or next patch release, depending on scope

## Disclosure

Please give the maintainer reasonable time to investigate and prepare a fix
before sharing full technical details publicly.
