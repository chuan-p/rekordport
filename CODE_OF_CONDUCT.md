# Code Of Conduct

## Our Pledge

We want `rekordport` to be a welcoming project for users and contributors of all
experience levels. We pledge to act and interact in ways that contribute to an
open, respectful, and harassment-free community.

## Our Standards

Examples of behavior that helps build a positive environment include:

- Showing empathy and patience when answering questions or reviewing changes
- Giving and receiving constructive feedback in good faith
- Respecting differing viewpoints, workflows, and levels of experience
- Taking responsibility for mistakes and learning from them
- Focusing on what is best for the project and its users

Examples of unacceptable behavior include:

- Harassment, intimidation, or personal attacks
- Trolling, insulting, or derogatory comments
- Publishing someone else's private information without permission
- Repeatedly derailing technical discussions
- Any conduct that would be inappropriate in a professional setting

## Scope

This Code of Conduct applies within project spaces, including GitHub issues,
pull requests, code review, and other public or private interactions that are
clearly connected to `rekordport`.

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of
Conduct and may remove, edit, or reject comments, commits, code, issues, and
other contributions that are not aligned with it.

## Reporting

If you experience or witness behavior that violates this Code of Conduct,
contact the maintainer through GitHub. If you are not comfortable sharing the
details in public, open a short issue that asks for a private moderation
follow-up without including sensitive details.

All reports will be reviewed promptly and handled with discretion.
